# FPH Exam – Swiss Specialist Pharmacist Exam Simulator

Mobile-first web app for practicing **FPH Officinal Pharmacy** (Swiss Specialist Pharmacist) exam simulations.

---

## Project Summary

### What was done
1. **Real exam analysis** — read the sample PDF (`completo_FPH Fragen April 2023-1.pdf`) to understand the format and difficulty of the original questions
2. **AI-generated questions** — used Google NotebookLM (Gemini) to generate 101 questions in Italian, based on Marialucia's study notebooks
3. **Validated JSON file** — `esame_fph_100_domande.json` (101 questions, 101 KB)
4. **Web app** — React/Vite simulator with interactive quiz, statistics, and answer review
5. **Automation scripts** — `genera_esame.py` + `config_esame.json` to regenerate the exam in ~10 minutes with a single command

### Key file structure

## How to Regenerate Questions

**Full command (fully automated, ~10 min):**
```bash
cd "/Users/gianmichele/Desktop/esame marialucia"
source venv/bin/activate
python3 fph-exam/genera_esame.py
```

**Regenerate only one batch:**
```bash
python3 fph-exam/genera_esame.py --ruolo ruolo4c
```

**Assemble without regenerating (merges existing files in `roles/`):**
```bash
python3 fph-exam/genera_esame.py --solo-assembla
```

**Use a different config (for an exam on other topics):**
```bash
python3 fph-exam/genera_esame.py --config fph-exam/config_esame_v2.json
```

> Questions are generated by **Google NotebookLM (Gemini)** based on documents loaded into Marialucia's notebooks — not from generic knowledge.

---

## How to Update the App with New Questions

1. Generate the new JSON with `genera_esame.py`
2. Replace `esame_fph_100_domande.json` with the generated file
3. Run the simulation script:
   ```bash
   python3 genera_simulazione.py
   ```
4. Publish to Netlify:
   ```bash
   git add src/data/tests/ && git commit -m "new simulation" && git push
   ```
   Netlify redeploys automatically.

## Exam Structure

| Role | Area | Questions |
|------|------|-----------|
| 1 | Prescription and treatment plan validation | 7 |

| 4 | Clinical Pharmacy | 50 |
| 5 | Anamnesis and therapy | 10 |

| 7 | Laboratory results | 7 |
| 8 | Emergency situations | 7 |
| 9 | Vaccinations and blood draws | 7 |
| | **Total** | **101** |

**Passing threshold: 67/101 (≈ 67%) — Time allowed: 3h 30min**

> Note: the original distribution sums to 101, not 100. The JSON file reflects this.

## Supported Question Types

| Type | Description | Answer format |
|------|-------------|---------------|
| `multipla_scelta` | 4 options (A/B/C/D), 1 correct | `"B"` |
| `kprim` | 4 statements (1/2/3/4), each True/False | `"VFFV"` |

## Tech Stack

- **Frontend:** React + Vite + Tailwind CSS + React Router
- **Data:** localStorage (no DB, no backend)
- **Question generation:** Google NotebookLM (Gemini) via CLI `notebooklm`
- **Deploy:** Netlify (auto-deploy on push to main)
